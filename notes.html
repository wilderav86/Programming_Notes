Documents: Head and Body

To publish a full HTML document you need:

  1. a < !DOCTYPE html > tag
    • This tells the browser that it is loading HTML
  2. an < html > tag
    • Html tags encompass all other tags
    • you can include "lang="en" to tell the browser to read with english language.
  3. a < head > tag. Head tags contain things like:
    • stylesheets in CSS
    • Script in javascript
    • metadata for search engines
  4. a < title > tag
    • the title tag is contained within the head tag
  5. a < body > tag
    • All other content is contained within the body tags

Styling

You can use inline styling to change the look of elements.

For instance, if i wanted to make everything in this < p > tag blue, I would add the inline tag: style="color: blue"; inside the opening P tag. This method can be repititious and is prone to errors.

You can add as many style attributes as you want as long as they are divided by a semicolon and inside the quotation marks.

Example: style="color: blue; text-align: center";

A more efficient way of styling is using Style elements

Style elements are used with selectors to style multiple elements at a time.

For instance, if you want to style all h2 elements with the same font you would create an opening and closing "style " tag that contains "h2 {font-style:"comic sans"}"

Selectors

Class selectors work the same way style elements use selectors to style multiple elements at a time. Say you want to change the color of everything contained in a single < p > tag, while leaving other < p > tags alone. You can insert class="class_name" into the < p > tag, then create a "stlye" tag containing .class_selector { color:'red'}

id selectors work in the same way, but only when a single element needs styling. instead of a "." to reference an id selector, it uses a "#"

You can also combine selectors by using descendant selectors. For instance, if you want to style all < em > tags within an < li > tag, you would use li em {color: red;}

Fonts

To style an element with a specific font, use font-family: font name;.

When choosing a font, it's important to choose fallback fonts incase the browser doesn't support your first font choice

For example: font-family:constantia, georgia, sans-serif: will tell the browser to display "georgia" if it can't find "constantia", and the generic font family, "sans-serif", if it can't find georgia.

Here are some common fonts you can try:

  • Helvetica
  • "Times New Roman"
    • Note: If a font contains spaces in it, we put the name inside quotes (to make sure the browswe sees it as a single font name, rather than multiple words.)
  • Courier
  • Palatino
  • Impact

You can also try out the five generic font families

  • serif
  • sans-serif
  • cursive
  • fantasy
  • monospace

Remember, when you use a generic font family, this tells the browser "get me this general kind of font". For example, if you tell it to use font-family: monospace, it might use Inconsolata or Courier New (because these are both categorized as monospace fonts).

Here are some other properties to further style your font:

  • font-size:
  • font-weight: bold
  • font-style: italic
  • text-decoration: underline

Note: We can combine all of this styling info into one declaration, by using the short-hand font: property. This can be very convenient!

What's not so convenient is that the values for the font property have to be in a certain specific order or they won't work. This is a great example of why it's important to get comfortable looking things up in the documentation.

If you do a Google search for "CSS font property" or "CSS font property order", one of the top hits you'll get is this handy page from the CSS-Tricks website,. It describes the common "gotchas" that you'll encounter with the font property.

Flexible Box Model, "Flexbox"



MDN web docs: basic concepts of flexbox

When working with flexbox you need to think in terms of two axes — the main axis and the cross axis. The main axis is defined by the flex-direction property, and the cross axis runs perpendicular to it. Everything we do with flexbox refers back to these axes, so it is worth understanding how they work from the outset.

The main axis is defined by flex direction, which has four possible values:

  • row
  • row-reverse
  • column
  • column-reverse

Should you choose row or row-reverse, your main axis will run along the row in the inline direction.

Choose column or column-reverse and your main axis will run from the top of the page to the bottom in the block direction.

The cross axis runs perpendicular to the main axis, therefore if your flex-direction (main axis) is set to row or row-reverse, the cross axis runs down the columns.

If your main axis is column or column-reverse, then the cross axis runs along the rows.
